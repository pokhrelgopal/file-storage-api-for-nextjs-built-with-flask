cloud_upload MediaMaster API

Secure, fast, and reliable file storage API. Upload, manage, and serve your media files with ease. Perfect for developers building applications that need robust file storage capabilities.

RESTful API API Key Authentication Multiple File Formats 100MB Max Size
Quick Start
Base URL: {{ request.host_url }} API Key: mediamaster-dev-key-12345

API Endpoints

POST /file

Upload a media file

Headers
X-API-Key: mediamaster-dev-key-12345
Content-Type: multipart/form-data
Parameters
  • file (required) - The file to upload
Supported Formats

PNG JPG JPEG WEBP GIF MP4 AVI MOV MKV WEBM

Show Example
cURL
curl -X POST {{ request.host_url }}file \
-H "X-API-Key: mediamaster-dev-key-12345" \
-F "file=@image.jpg"
JavaScript
const formData = new FormData();
formData.append('file', fileInput.files[0]);

fetch('{{ request.host_url }}file', {
method: 'POST',
headers: {
'X-API-Key': 'mediamaster-dev-key-12345'
},
body: formData
})
.then(response => response.json())
.then(data => console.log(data));

GET /files

Get a list of uploaded files

Headers
X-API-Key: mediamaster-dev-key-12345
Query Parameters
  • page (optional) - Page number (default: 1)
  • per_page (optional) - Items per page (default: 20, max: 100)
Show Example
cURL
curl -X GET "{{ request.host_url }}files?page=1&per_page=10" \
-H "X-API-Key: mediamaster-dev-key-12345"
GET /file/{id}/info

Get information about a specific file

Headers
X-API-Key: mediamaster-dev-key-12345
Path Parameters
  • id (required) - File ID
DELETE /file/{id}

Delete a file (soft delete)

Headers
X-API-Key: mediamaster-dev-key-12345
Path Parameters
  • id (required) - File ID
GET /media/uploads/{filename}

Serve uploaded media files (No API key required)

Path Parameters
  • filename (required) - The filename to serve
GET /health

Health check endpoint (No API key required)

Returns service status and timestamp

key Authentication

All API endpoints (except health check and media serving) require authentication using an API key.

How to use:
  1. Include the API key in the request header
  2. Header name: X-API-Key
  3. Development key: mediamaster-dev-key-12345
Note: Change the API key in production environment via the API_KEY environment variable.
info Response Codes
200 Success
201 Created
400 Bad Request
401 Unauthorized
404 Not Found
413 File Too Large
500 Server Error
rule Limits & Rules
  • Max file size: 100MB
  • Supported formats: Images (PNG, JPG, JPEG, WEBP, GIF) and Videos (MP4, AVI, MOV, MKV, WEBM)
  • File naming: Files are automatically renamed with secure random prefixes
  • Storage: Soft delete - files are marked as inactive but not physically removed
